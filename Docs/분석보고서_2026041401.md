# [분석보고서] DotCamera .dmg 실행 오류 분석

@>-- 2026:04:14

---

## 1. 개요
DotCamera 프로젝트의 Electron 빌드 결과물인 `.dmg` 파일(v1.0.4 기준)을 macOS에서 실행했을 때, 실시간 필터가 정상적으로 작동하지 않고 화면이 흰색 또는 검은색으로 고정되는 현상이 발생함.

### 1.1 주요 증상
- **화면 출력 오류**: 웹캠 영상이 필터링되어 나타나지 않고 단색 배경만 표시됨.
- **상태 표시 미흡**: 시스템 타스크바에 카메라 사용 중임을 알리는 표시(녹색 점 등)가 나타나지 않음.
- **초기화 실패**: 비디오 메타데이터를 읽어오지 못해 렌더링 해상도가 0으로 고정됨.

---

## 2. 원인 분석

| 구분 | 세부 원인 및 분석 내용 |
|------|----------------------|
| **메타데이터 로드 실패** | `video.onloadedmetadata` 이벤트가 Electron 렌더러 프로세스에서 간헐적으로 발생하지 않음. 이로 인해 `videoWidth`, `videoHeight`가 0으로 인식되어 텍스처 초기화에 실패함. |
| **중간 데이터 누실** | 기존 방식은 `video` -> `Canvas 2D` -> `ImageData` -> `SwissGL Texture` 단계를 거침. `getImageData()` 호출 시 비디오가 준비되지 않아 빈(Empty) 데이터를 반환하거나 성능 저하로 프레임 드랍 발생. |
| **권한 핸들러 미설정** | `electron_main.js`에서 카메라 및 마이크 접근 권한에 대한 `setPermissionCheckHandler`가 명시적으로 설정되지 않아 샌드박스 환경에서 접근이 차단됨. |
| **저해상도 고정** | SwissGL 텍스처 생성 시 해상도가 160x96으로 고정되어 있어, 고해상도 웹캠 입력 시 필터 효과가 뭉개지거나 거의 보이지 않음. |

## 3. 해결 방안 (Technical Solutions)

### 3.1 렌더링 파이프라인 최적화 (Direct Input)
Canvas 2D를 거치지 않고 `video` 요소를 SwissGL에 직접 전달하여 메모리 복사를 줄이고 데이터 무결성을 확보함.
```javascript
// 수정 전: Canvas 2D 사용
// videoCtx.drawImage(this.video, 0, 0, w, h);
// const imageData = videoCtx.getImageData(0, 0, w, h);
// const tex = glsl({}, {data:imageData.data, ...});

// 수정 후: video 직접 입력
const tex = glsl({}, { data: this.video, tag: 'video' });
```

### 3.2 상태 체크 로직 강화
비디오 데이터가 완전히 로드되었는지 확인하는 `readyState` 체크를 추가하여 초기화 시점을 보장함.
```javascript
if (this.video.readyState < HTMLMediaElement.HAVE_ENOUGH_DATA) {
    return; // 데이터 준비될 때까지 렌더링 유예
}
```

### 3.3 Electron 메인 프로세스 보안 설정 수정
`electron_main.js`에 세션 권한 핸들러를 추가하여 앱 실행 시 권한 요청을 명시적으로 허용함.
```javascript
// electron_main.js
win.webContents.session.setPermissionCheckHandler((webContents, permission) => {
  if (permission === 'media') return true;
  return false;
});
```
