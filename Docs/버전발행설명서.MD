## 1. 개요

DotCamera Desktop App의 버전을 관리하고 발행하는 방법에 대한 설명서입니다.

---

## 2. 버전 규칙

### 버전 형식
```
v{major}.{minor}.{patch}
```

| 구분 | 의미 | 예시 |
|------|------|------|
| **major** | 주요 변경 (호환성 중단) | 1.0.7 → 2.0.0 |
| **minor** | 기능 추가 (하위 호환) | 1.0.7 → 1.1.0 |
| **patch** | 버그 수정 (하위 호환) | 1.0.7 → 1.0.8 |

---

## 3. 빌드 및 버전 발행

### 방법 1: 자동 버전 증가 (권장)

```bash
./build.sh
```

**작업:**
1. package.json 버전에서 patch 번호 1 증가
2. `npm run build` 실행
3. DMG 파일 생성

**출력:**
```
dist/DotCamera-1.0.8.dmg
```

### 방법 2: 수동 버전 설정

```bash
# package.json 편집
vi package.json
# version: "1.0.7" → "1.0.8"
# 저장 후

# 빌드
npm run build
```

### 방법 3: 특정 버전으로 빌드

```bash
# 버전指定
npm version 1.0.8

# 빌드
npm run build
```

### 생성 파일

## 6.发行 프로세스

### 전체 플로우

```
1. 코드 수정
2. 테스트 (npm start)
3. 버전 결정 (patch/minor/major)
4. ./build.sh 실행
5. DMG 생성 확인
6. Git 커밋 및 푸시
7. GitHub Release (선택)
```

### Git 커밋 및 푸시

```bash
# 변경사항 확인
git status

# 커밋
git add .
git commit -m "Release v1.0.8"

# 푸시
git push origin main
```

## 7. 자주 사용하는 명령어

| 명령어 | 용도 |
|--------|------|
| `./build.sh` | 버전 증가 + 빌드 |
| `npm start` | 개발 모드 실행 |
| `npm run build` | 빌드만 수행 |
| `npm version patch` | 패치 버전 1 증가 |
| `npm version minor` | 마이너 버전 1 증가 |
| `npm version major` | 메이저 버전 1 증가 |

## 9. 빌드 설정

### package.json build 섹션

```json
{
  "build": {
    "appId": "com.atom.dotcamera",
    "productName": "DotCamera",
    "mac": {
      "target": "dmg",
      "category": "public.app-category.photography",
      "entitlements": "build/entitlements.mac.plist"
    }
  }
}
```

### entitlements 설정

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>com.apple.security.device.camera</key>
    <true/>
    <key>com.apple.security.device.audio-input</key>
    <true/>
    <key>com.apple.security.cs.allow-jit</key>
    <true/>
</dict>
</plist>
```
