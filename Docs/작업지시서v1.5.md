## [2] 코딩 규칙

- **작성자**: `@>-- microdit` 로 기록
- **수정일자**: 수정한 위치 위에 `@>-- yyyy:mm:dd` 기록
- **주석**: 수정하는 내용 최대한 주석으로 상세하게 설명

### Docstring 필수
수정/생성하는 파일 최상단에 Docstring 작성:
- 파일명
- 작업시간: yyyy:mm:dd HH:MM
- 분석된 현재 코드의 핵심 내용
- 실행에 필요한 방법
- 연관된 프로그램이 있을 경우 구성도 (tree 형태)

### 로그 추가

- 기존 로그 파일에 append, 파일명과 형식 유지

## [3] 작업 규칙
- **프로젝트 시작해**: 현재 프로젝트는 새로 만들어진 것으로 무엇을 만들고자 하는지 용도를 분석한다. 그리고 현재의 문서 양식들을 참고해서 오늘 일자 기준으로 프로젝트 목표와 목적에 맞게 새로 구성한다.
- **"작업하자"**: 진행된 내용과 미진한 내용을 분석하고 보고한다. 현재 작업지시서 버전과 작업위치 정보를 정확하게 보고한다.
- **"버전 올려"**: 현재 작업된 내용들을 모든 보고서에 기록한다. git 커밋 후 푸시까지 완료되면 보고한다.
- **중복 실행 방지**: 작업 전 먼저 설명하고 허락받고 진행한다.
- **경로 접근 제한**: 현재의 작업공간 이외 경로 접근 금지
- **접근 허가 필요**: 현재의 작업공간 이외 접근이 필요한 경우 작업 목적 설명하고 허락받고 진행
- **보고서 작성해**: 현재 일자 기준으로 WORK_LOG*.MD ,  TODOLIST_REPORT*.MD, 설계서*.MD, 사용설명서*.MD, *가이드*.MD, SESSION*.LOG, AGENTS.MD 파일을 모두 최신 정보로 생성 또는 갱신한다. 생성하는 파일 또는 갱신하는 파일명에는 오늘 일자를 확장자 앞부분에 포함시켜 저장 관리한다.
파일 및 폴더가 바뀐 경우 구조를 반영하여 아키텍처를 작성한다. BATCH 파일이 생성되거나 수정된 경우 필히 모두 최신정보로 사용법을 생성 또는 갱신한다. 모든 파일들을 기록 했으면 GIT 커밋 및 GitHub에 푸시 한다. 
- **자체 검수해**: 개발한 코드에 대해서 스스로 검증한다. 테스트 직접 수행한다. 테스트에 입력과 출력을 확인하고 보고한다. 자체 테스트 결과가 확인되면 검수 내용을 보고한다. 자체 검수에 이상이 있을 스스로 수정하고 테스트하고 목적한 결과가 나오면 이를 보고한다.
- **깃작업해**: DEV 에 GIT 커밋 및 푸시, PRD 에 GIT 커밋 및 푸시, GitHub 에 푸시하고 결과를 내용을 보고한다.

### Git 관련 규칙
- git server 접근: 허락을 받아서 한다
- git server 커밋: 사전에 보고하고 허락을 받아서 한다
- 공개문서 폴더와 비공개문서 폴더를 구별
