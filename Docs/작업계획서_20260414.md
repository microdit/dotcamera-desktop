## 2. 기능 설명

### 2.1 현재 구현된 기능
1. **웹캠 캡처** - `navigator.mediaDevices.getUserMedia()` 사용
2. ** видео 프레임 추출** - Canvas 2D API로 video → ImageData 변환
3. **Dot 필터 효과** - SwissGL로 웹캠 영상에 도트 패턴 필터 적용
4. **Electron 데스크톱 앱** - .dmg 패키징

### 2.2 필터 알고리즘 (dotcamera.html)
```
1. video → Canvas 2D (ImageData)
2. ImageData → SwissGL Texture (tex)
3. tex → 밝기 계산 (lum) → 멀티스케일 피라미드 (lum0~6)
4. 피라미드 → 그래디언트 (grad)
5. grad → 파티클 시뮬레이션 (points)
6. points → 화면 렌더링
```

## 4. 현재 문제점

### 4.1 증상
- 화면이 **흰색/검정색**만 표시
- 웹캠 영상이 **필터에 반영되지 않음**
- 타스크바에 웹캠 사용 표시 안됨

### 4.2 로그 분석 (v1.0.4)

```
✓ Camera stream OK        # 웹캠 스트림 생성 성공
✓ Video play started     # 비디오 재생 시작
✗ Video metadata 미출력   # onloadedmetadata 이벤트 미발생
✗ Video dims 미출력      # videoWidth/videoHeight = 0
✗ "Processing video" 미출력 # calcForceField에서 프레임 처리 안됨
✓ resized tex (160,96)   # SwissGL 텍스처 생성 (해상도 낮음: 160x96)
```

### 4.3 원인 분석

| 구분 | 분석 |
|------|------|
| **웹캠 권한** | ✓ 정상 (Camera stream OK) |
| **비디오 재생** | ✓ 정상 (Video play started) |
| **비디오 상태** | ✗ `videoWidth=0, videoHeight=0` - 메타데이터 로드 안됨 |
| **SwissGL 텍스처** | ✓ 생성됨 (160x96) -videoCanvas에서 ImageData 추출 |
| **화면 출력** | ✗ 흰색 도트만 보임 - 웹캠 영상 기반 필터 아님 |

**주요 원인:**
1. **비디오 메타데이터 로드 실패** - Electron 환경에서 `onloadedmetadata` 이벤트가 제대로 발생하지 않음
2. **Canvas 2D → SwissGL 데이터 전달 문제** - `getImageData()`가 빈 데이터 반환 가능성
3. **텍스처 해상도 문제** - 160x96으로 너무 낮아 필터 효과가不明显

## 5. 수정 방안

### 5.1方案 1: 직접 video → SwissGL 텍스처

```javascript
// 기존: Canvas 2D 우회
videoCtx.drawImage(this.video, 0, 0, w, h);
const imageData = videoCtx.getImageData(0, 0, w, h);
const tex = glsl({}, {size:[w, h], data:imageData.data, tag:'video'});

// 수정: video 요소를 직접 전달
const tex = glsl({}, {size:[this.video.videoWidth, this.video.videoHeight], data:this.video, tag:'video'});
```

### 5.2方案 2: video.readyState 체크 강화

```javascript
// 프레임 캡처 전 충분한 데이터 로드 대기
if (this.video.readyState < HTMLMediaElement.HAVE_ENOUGH_DATA) {
    return this.tmp;
}
```

### 5.3方案 3: Electron 권한 추가 설정

```javascript
// electron_main.js에 추가
win.webContents.session.setPermissionCheckHandler(() => true);
```

## 6. 구현 계획

### 6.1 수정 순서

1. **1단계:** video 요소 직접 SwissGL에 전달 (Canvas 2D 제거)

3. **3단계:** 디버그 로그 제거 및 최적화
